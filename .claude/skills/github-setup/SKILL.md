# Skill: github-setup

Create GitHub repo(s) for the project and configure initial Git structure.

## Prerequisites
- `gh` CLI authenticated (GITHUB_TOKEN in environment)
- `git` available
- Project workspace exists at $PROJECT_DIR

## Decision: mono-repo vs multi-repo

Use **mono-repo** when:
- Single team
- Frontend and backend closely coupled
- Shared types, contracts, or generated code
- Small-to-medium project scope

Use **multi-repo** when:
- Multiple teams with independent deploy cycles
- Frontend and backend have different tech stacks or release cadences
- Microservices architecture
- Teams explicitly need independent CI/CD pipelines

Document the decision with rationale in `.agentflow/architect/specs/architecture.md`
under a `## Repository Structure` section.

## Steps

### 1. Determine structure
Read `.agentflow/pm/setup.md` and `.agentflow/architect/setup/teams.md`.
Apply decision criteria above. Write decision to `architecture.md`.

### 2a. Mono-repo setup
```bash
# Create repo
gh repo create $GITHUB_ORG/$PROJECT_NAME \
  --private \
  --description "[project description from briefing]" \
  --clone=false

# Init local git
cd $PROJECT_DIR
git init
git remote add origin https://github.com/$GITHUB_ORG/$PROJECT_NAME.git

# Write .gitignore
cat > .gitignore << 'EOF'
# AgentFlow internals
.agentflow/.log-*

# OS
.DS_Store
Thumbs.db

# Env
.env
.env.*
!.env.example

# Node
node_modules/
dist/
build/

# Python
__pycache__/
*.pyc
.pytest_cache/
venv/

# IDE
.vscode/
.idea/
EOF

# Write initial README (Architect updates this at each milestone review)
cat > README.md << 'EOF'
# $PROJECT_NAME

> Managed by AgentFlow multi-agent framework.

## Status
🚧 Setup phase — detailed README will be generated by Architect at first milestone.

## Branches
- `main` — stable, merged sprints only
- `sprint-NN` — active sprint work

## Structure
```
.agentflow/architect/specs/    API, schema, UI specs (source of truth)
.agentflow/teams/              Implementation per team
.agentflow/pm/                 Sprint planning and status
.agentflow/decisions/          User decisions and notifications
```
EOF

# Initial commit on main
git add .
git commit -m "chore: initial project structure [agentflow]"
git branch -M main
git push -u origin main

# Create sprint-01 branch
git checkout -b sprint-01
git push -u origin sprint-01
```

### 2b. Multi-repo setup
Create separate repos for each component. Typical split:
- `$PROJECT_NAME-frontend`
- `$PROJECT_NAME-backend`
- `$PROJECT_NAME-specs` (architecture docs, api.yaml, schema.sql)

```bash
for component in frontend backend specs; do
  gh repo create $GITHUB_ORG/$PROJECT_NAME-$component \
    --private \
    --description "$PROJECT_NAME — $component"
done
```

Then init each repo with its own .gitignore and README,
push initial commit, and create sprint-01 branch in each.

### 3. Store repo info in .agentflow/CONTEXT.md
After creating repos, update `.agentflow/CONTEXT.md`:

```yaml
github:
  org: $GITHUB_ORG
  structure: mono|multi
  repos:
    - name: $PROJECT_NAME          # mono-repo
      url: https://github.com/...
      current_branch: sprint-01
  # or for multi:
    - name: $PROJECT_NAME-frontend
      url: https://github.com/...
      current_branch: sprint-01
    - name: $PROJECT_NAME-backend
      ...
    - name: $PROJECT_NAME-specs
      ...
```

### 4. Notify lead
Send a message to the lead confirming that repos are configured.
Include repo URLs for each team to clone and the sprint branch name.

## Continue mode (existing codebase)

When `.agentflow/CONTEXT.md` has `mode: continue`:
1. **Skip repo creation** — the repo already exists
2. Read `github.repos` from .agentflow/CONTEXT.md — remote URL is pre-populated from the existing repo
3. Verify remote is reachable: `git -C $PROJECT_DIR remote -v`
4. Create sprint branch from current HEAD:
   ```bash
   cd $PROJECT_DIR
   git checkout -b sprint-01
   git push -u origin sprint-01
   ```
5. Update .agentflow/CONTEXT.md `current_branch: sprint-01`
6. Do NOT create .gitignore or README.md — the repo already has them

## Error handling
- If `gh` not authenticated → write to issues: "GITHUB_TOKEN not set.
  Set env var and re-run architect setup."
- If repo name taken → append `-app` or ask PM to escalate to user
- Always verify push succeeded before updating .agentflow/CONTEXT.md
